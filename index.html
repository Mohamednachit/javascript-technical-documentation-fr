Introduction

JavaScript est un langage de programmation qui permet de rendre les pages web interactives. Contrairement au HTML (structure) et au CSS (mise en forme), JavaScript est utilisé pour ajouter du comportement dynamique à une page.

C'est un langage :

  • Interprété : il est exécuté directement par le navigateur, sans besoin de compilation.
  • Multi-paradigme : il supporte la programmation impérative, fonctionnelle et orientée objet.
  • Basé sur des prototypes : l'héritage en JavaScript se fait par clonage d'objets existants (et non par classes comme en Java classique, bien que les classes soient disponibles depuis ES6).
  • Dynamique : les types de variables peuvent changer à l'exécution.

🎯 À quoi sert JavaScript ?

Voici quelques exemples d'usages courants :

  • 💡 Modifier dynamiquement le contenu HTML d'une page (par exemple : afficher un message ou des données).
  • 🎨 Changer le style CSS d'un élément sans recharger la page (par exemple : changer une couleur ou une taille au clic).
  • 🖱️ Réagir aux actions de l'utilisateur (clic, survol, clavier, etc.).

📌 Exemple de code simple :

Ce code modifie le contenu d’un élément HTML avec l’ID demo :


        // Ceci est un commentaire en JavaScript
        document.getElementById("demo").innerHTML = "Bonjour JavaScript!";

➡️ Ce que fait ce code :

  • document : fait référence à la page HTML.
  • getElementById("demo") : sélectionne l’élément dont l’ID est "demo".
  • innerHTML = "Bonjour JavaScript!" : remplace le contenu HTML de cet élément.

🔍 Remarque importante : JavaScript n’est pas Java. Ce sont deux langages totalement différents :

  • Java est un langage compilé orienté objet (souvent utilisé côté serveur ou dans les applications Android).
  • JavaScript est principalement utilisé dans le navigateur, côté client, pour rendre les pages interactives.

📅 JavaScript a été créé en 1995 par Brendan Eich en seulement 10 jours, initialement sous le nom de Mocha, puis renommé en LiveScript, avant d'être finalement appelé JavaScript pour des raisons marketing.

Syntaxe de base

La syntaxe en JavaScript désigne l’ensemble des règles qui permettent d’écrire un code compréhensible et exécutable par le navigateur. Voici les principaux éléments à connaître :

  • 📍 Les instructions se terminent généralement par un point-virgule ;. Bien qu’il soit parfois optionnel, l’utiliser évite des erreurs inattendues.
  • 📦 Les blocs de code (par exemple dans une fonction ou une condition) sont entourés d’accolades { }.
  • 📝 Les variables peuvent être déclarées avec :
    • var : ancienne méthode, avec un comportement de portée fonctionnelle.
    • let : pour déclarer des variables dont la valeur peut changer (portée bloc).
    • const : pour déclarer des constantes (valeurs qui ne doivent pas être réassignées).

📌 Exemple de syntaxe de base

Voici un petit script illustrant ces règles :


        // Déclaration d'une variable avec let
        let message = "Hello";
        
        // Déclaration d'une fonction
        function showMessage() {
            // Affiche une boîte de dialogue avec le message
            alert(message);
        }
        
        // Appel de la fonction
        showMessage();

📎 Ce code fait trois choses :

  1. Déclare une variable nommée message contenant la chaîne "Hello".
  2. Crée une fonction showMessage() qui utilise la méthode alert() pour afficher cette valeur.
  3. Appelle la fonction pour qu’elle s’exécute.

⚠️ Sensibilité à la casse

JavaScript est sensible à la casse (case-sensitive), ce qui signifie que Message et message sont deux noms différents. Écrivez vos variables et fonctions avec rigueur.

🔤 Littéraux (valeurs directes)

En JavaScript, vous pouvez utiliser différents types de littéraux pour représenter des données simples :


        // Nombres
        123         // entier
        3.14        // nombre à virgule flottante
        
        // Chaînes de caractères
        "Bonjour"   // guillemets doubles
        'Hello'     // ou simples
        
        // Booléens
        true
        false
        
        // Tableaux (collections de valeurs indexées)
        [1, 2, 3]
        
        // Objets (collections de paires clé:valeur)
        {nom: "Jean", age: 30}

Ces types de données sont les fondations de la manipulation d’informations en JavaScript.

Variables et constantes

Les variables servent à stocker des données (nombres, chaînes, objets, etc.) dans un programme. En JavaScript, il existe trois façons de les déclarer :

  1. let – déclare une variable modifiable, avec une portée limitée au bloc (entre accolades).
  2. const – déclare une constante, dont la valeur ne peut pas être réassignée.
  3. var – ancienne façon de déclarer une variable, avec une portée fonctionnelle. Elle est déconseillée dans les nouveaux projets.

🧪 Exemples de déclarations :


          // Déclaration avec let : la valeur peut être modifiée
          let age = 25;
          age = 26; // ✅ Modification autorisée
          
          // Déclaration avec const : la valeur est fixe
          const PI = 3.14;
          // PI = 3.1415; // ❌ Erreur ! Impossible de modifier une constante
          
          // Déclaration avec var : fonctionnelle, à éviter
          var nom = "Jean";

✏️ Règles de nommage des variables :

En JavaScript, un nom de variable (ou d’identifiant) doit respecter certaines règles :

  • 🔤 Doit commencer par une lettre (a-z ou A-Z), un underscore (_) ou un dollar ($)
  • 🔢 Peut contenir des chiffres, mais ne peut pas commencer par un chiffre
  • 🧠 Est sensible à la casse : maVariable ≠ MAVariable
  • 🚫 Ne doit pas être un mot réservé de JavaScript (comme if, while, let, etc.)

✅ Bonnes pratiques de nommage :

Adopter un style clair et cohérent améliore la lisibilité de votre code :


          // camelCase pour les noms de variables
          let monNomComplet = "Jean Dupont";
          
          // UPPER_CASE pour les constantes
          const TAUX_TVA = 0.2;
          
          // Des noms explicites et descriptifs
          let nombreUtilisateursConnectes = 100;

💡 Remarques importantes :

  • On privilégiera let et const pour tous les nouveaux projets.
  • const ne signifie pas que l’objet est immuable, mais que la référence ne peut pas changer. Exemple :

          const utilisateur = { nom: "Alice" };
          utilisateur.nom = "Bob"; // ✅ Ceci est autorisé
          // utilisateur = { nom: "Charlie" }; // ❌ Ceci génère une erreur

Comprendre la portée (scope) des variables est également essentiel, mais cela fera l’objet d’une autre section dédiée.

Structures de contrôle

Les structures de contrôle permettent de gérer le flux d'exécution d'un programme : conditions, répétitions, et choix multiples.

🧭 Structures conditionnelles

Permettent d’exécuter du code selon que des conditions soient vraies ou non :


          // if...else
          let age = 17;
          
          if (age >= 18) {
              console.log("Majeur");
          } else {
              console.log("Mineur");
          }
          
          // if...else if...else
          let temperature = 15;
          
          if (temperature > 30) {
              console.log("Très chaud");
          } else if (temperature > 20) {
              console.log("Chaud");
          } else {
              console.log("Frais");
          }

🧩 switch : alternative à plusieurs if

Utile pour tester plusieurs cas possibles :


          let note = 'B';
          
          switch (note) {
              case 'A':
                  console.log("Excellent");
                  break;
              case 'B':
                  console.log("Bien");
                  break;
              case 'C':
                  console.log("Moyen");
                  break;
              default:
                  console.log("Non évalué");
          }

🔁 Boucles (répétition de blocs)

1. for : boucle à compteur


          for (let i = 0; i < 5; i++) {
              console.log("i vaut : " + i);
          }

2. while : boucle tant qu’une condition est vraie


          let i = 0;
          
          while (i < 5) {
              console.log("i vaut : " + i);
              i++;
          }

3. do...while : boucle exécutée au moins une fois


          let j = 0;
          
          do {
              console.log("j vaut : " + j);
              j++;
          } while (j < 5);

🔎 Opérateurs de comparaison

Ces opérateurs sont utilisés dans les conditions :

  • == : égalité avec conversion de type (ex : '5' == 5 ➜ true)
  • === : égalité stricte (même type et même valeur)
  • != : inégalité (conversion de type)
  • !== : inégalité stricte
  • >, <, >=, <= : comparaisons classiques

💡 Exemple pratique :


          let a = '5';
          let b = 5;
          
          console.log(a == b);   // true (valeurs équivalentes)
          console.log(a === b);  // false (types différents)

Utilisez === et !== dans vos projets pour éviter les erreurs liées aux conversions implicites de type.

Fonctions

Les fonctions sont des blocs de code conçus pour effectuer une tâche spécifique. Elles permettent de réutiliser du code sans le réécrire plusieurs fois.

📌 Déclaration de fonction (function declaration)

Déclare une fonction nommée. Elle peut être appelée avant sa déclaration grâce au "hoisting".


          function carre(x) {
              return x * x;
          }
          
          console.log(carre(5)); // Affiche 25

📌 Expression de fonction (function expression)

Attribue une fonction anonyme à une variable. Cette fonction ne peut être appelée qu'après sa déclaration.


          const carre = function(x) {
              return x * x;
          };
          
          console.log(carre(4)); // Affiche 16

📌 Fonction fléchée (arrow function - ES6)

Synthaxe moderne et plus concise. Ne possède pas de this propre, utile dans les fonctions de rappel (callbacks).


          const carre = x => x * x;
          
          // Avec plusieurs instructions
          const somme = (a, b) => {
              const resultat = a + b;
              return resultat;
          };
          
          console.log(somme(3, 7)); // Affiche 10

📥 Paramètres et arguments

Les fonctions peuvent recevoir des paramètres et retourner des résultats. Des valeurs par défaut peuvent être définies.


          function afficherInfos(nom, age = 20) {
              console.log(`Nom: ${nom}, Age: ${age}`);
          }
          
          afficherInfos("Jean");         // Affiche: Nom: Jean, Age: 20
          afficherInfos("Marie", 25);    // Affiche: Nom: Marie, Age: 25

🔄 Fonctions d'ordre supérieur

Ce sont des fonctions qui prennent une ou plusieurs fonctions en paramètre, ou qui en retournent une.


          // Fonction qui applique une opération
          function appliquerOperation(x, operation) {
              return operation(x);
          }
          
          // Appel avec une fonction fléchée
          const resultat = appliquerOperation(5, x => x * x);
          console.log(resultat); // Affiche 25

✅ Bonnes pratiques

  • Utiliser des noms clairs et descriptifs (ex: calculerTotal, afficherMessage)
  • Préférer les fonctions fléchées pour les callbacks simples
  • Utiliser des valeurs par défaut pour éviter les erreurs de paramètres manquants
  • Modulariser votre code en petites fonctions réutilisables

Objets

En JavaScript, un objet est une structure permettant de regrouper des données (appelées propriétés) et des comportements (appelés méthodes) sous une même entité.

Les objets sont créés à l'aide de paires clé: valeur.

🔸 Déclaration d’un objet


          const personne = {
              nom: "Jean",
              age: 30,
              estActif: true
          };

🔹 Accès aux propriétés

  • objet.propriete
  • objet["propriete"] (utile pour les noms dynamiques ou contenant des espaces)

          console.log(personne.nom);       // "Jean"
          console.log(personne["age"]);   // 30

🔧 Modification et ajout de propriétés


          personne.age = 31;               // Modification
          personne.email = "jean@mail.com"; // Ajout

⚙️ Méthodes (fonctions dans un objet)

Une méthode est une fonction associée à un objet.


          const voiture = {
              marque: "Toyota",
              demarrer: function() {
                  console.log("La voiture démarre");
              }
          };
          
          voiture.demarrer(); // Appel de méthode

🧠 Le mot-clé this

Dans une méthode, this fait référence à l'objet lui-même.


          const utilisateur = {
              prenom: "Alice",
              saluer: function() {
                  console.log("Bonjour, je suis " + this.prenom);
              }
          };
          
          utilisateur.saluer(); // Bonjour, je suis Alice

🔁 Parcourir un objet


          for (let cle in personne) {
              console.log(cle + ": " + personne[cle]);
          }

✅ Bonnes pratiques

  • Utiliser const pour créer des objets que vous ne redéfinirez pas
  • Organiser les données liées ensemble dans un même objet
  • Utiliser des noms clairs pour les propriétés et méthodes

Les objets sont la base de JavaScript : tout (ou presque) est un objet, y compris les fonctions et les tableaux.

Tableaux

Un tableau (array) est une structure de données utilisée pour stocker plusieurs valeurs dans une seule variable. Les éléments sont ordonnés et indexés à partir de 0.

🔸 Déclaration d’un tableau


          // Tableau de chaînes
          const fruits = ["pomme", "banane", "orange"];
          
          // Tableau de nombres
          const nombres = [10, 20, 30];
          
          // Tableau mixte
          const melange = ["texte", 42, true];

🔹 Accès et modification


          console.log(fruits[0]);   // "pomme"
          fruits[1] = "kiwi";       // Remplace "banane" par "kiwi"
          console.log(fruits);      // ["pomme", "kiwi", "orange"]

➕ Ajouter et supprimer des éléments


          fruits.push("ananas");       // Ajoute à la fin
          fruits.unshift("mangue");    // Ajoute au début
          
          fruits.pop();                // Supprime le dernier
          fruits.shift();              // Supprime le premier

🔁 Parcourir un tableau


          // for classique
          for (let i = 0; i < fruits.length; i++) {
              console.log(fruits[i]);
          }
          
          // for...of
          for (const fruit of fruits) {
              console.log(fruit);
          }

🔧 Méthodes utiles


          fruits.includes("pomme");      // true
          fruits.indexOf("orange");      // 2
          fruits.splice(1, 1);           // Supprime 1 élément à l'index 1
          fruits.sort();                 // Trie alphabétiquement
          fruits.reverse();              // Inverse l'ordre

💡 Bonnes pratiques

  • Utiliser const si vous ne redéclarez pas le tableau
  • Éviter les "trous" (indices manquants)
  • Utiliser for...of ou forEach() pour une meilleure lisibilité

📦 Méthodes avancées (ES6+)


          const fruitsEnMajuscules = fruits.map(fruit => fruit.toUpperCase());
          const fruitsFiltres = fruits.filter(fruit => fruit.length > 5);
          const total = nombres.reduce((acc, val) => acc + val, 0);

Les tableaux sont largement utilisés en JavaScript pour manipuler des listes de données, créer des interfaces dynamiques, stocker des résultats, etc.

Manipulation DOM

Le DOM (Document Object Model) est une représentation en mémoire de la structure HTML d’une page web. Grâce à JavaScript, on peut accéder, modifier ou supprimer dynamiquement des éléments du DOM.

🔍 Accès aux éléments


          document.getElementById("demo");         // Par ID
          document.getElementsByClassName("box");  // Par classe (HTMLCollection)
          document.getElementsByTagName("p");      // Par balise
          document.querySelector(".box");          // Premier élément correspondant au sélecteur CSS
          document.querySelectorAll(".box");       // Tous les éléments correspondants

✏️ Modifier le contenu


          document.getElementById("demo").textContent = "Bonjour";  // Modifie le texte
          document.getElementById("demo").innerHTML = "<strong>Salut</strong>"; // Interprète HTML

🎨 Modifier le style


          const elem = document.getElementById("demo");
          elem.style.color = "blue";
          elem.style.backgroundColor = "#f0f0f0";

🧩 Ajouter/Supprimer des éléments


          // Créer un nouvel élément
          const newPara = document.createElement("p");
          newPara.textContent = "Texte ajouté dynamiquement";
          document.body.appendChild(newPara); // Ajout à la fin du 
          
          // Supprimer un élément
          const demo = document.getElementById("demo");
          demo.remove();

🧠 Gérer les attributs


          const image = document.getElementById("photo");
          image.setAttribute("src", "image.jpg");
          image.getAttribute("src");              // Lire un attribut
          image.removeAttribute("alt");           // Supprimer un attribut

🎯 Écouter des événements


          const btn = document.getElementById("btn");
          btn.addEventListener("click", () => {
              alert("Bouton cliqué !");
          });

💡 Bonnes pratiques

  • Utiliser querySelector et querySelectorAll pour plus de flexibilité
  • Minimiser les modifications directes dans le DOM pour optimiser les performances
  • Éviter d'utiliser innerHTML si du contenu HTML externe est injecté (risque XSS)

La manipulation du DOM est essentielle pour rendre les pages web interactives (comme cacher/afficher du contenu, modifier dynamiquement les textes, gérer des animations, etc.).

Événements

JavaScript permet de réagir aux événements déclenchés par l'utilisateur, tels que les clics, les saisies au clavier, les mouvements de la souris, etc. Grâce aux événements, vous pouvez rendre vos pages web interactives.

💡 Qu'est-ce qu'un événement ?

Un événement est une action détectée par le navigateur, comme un clic de souris, un survol de souris, une pression sur une touche, ou encore le chargement d'une page. Vous pouvez écouter ces événements et réagir en conséquence avec des fonctions JavaScript.

🔑 Déclaration d'un gestionnaire d'événements

Vous pouvez associer un événement à un élément HTML en utilisant la méthode addEventListener(). Cela permet de définir un comportement spécifique lorsque l'événement se produit.


          const button = document.getElementById("btn");
          
          // Écouter l'événement "click" sur un bouton
          button.addEventListener("click", () => alert("Cliqué!"));

Dans l'exemple ci-dessus, chaque fois que l'utilisateur clique sur le bouton avec l'ID btn, l'alerte "Cliqué!" s'affiche.

💻 Types d'événements courants

  • click : Lorsqu'un utilisateur clique sur un élément (par exemple, un bouton)
  • keydown : Lorsqu'une touche du clavier est enfoncée
  • keyup : Lorsqu'une touche du clavier est relâchée
  • mousemove : Lorsqu'un mouvement de souris est détecté
  • submit : Lorsqu'un formulaire est soumis
  • focus : Lorsqu'un élément obtient le focus
  • blur : Lorsqu'un élément perd le focus

💡 Exemple : Événements sur le clavier


          const input = document.getElementById("texte");
          
          input.addEventListener("keydown", (event) => {
              console.log("Touche enfoncée : " + event.key);
          });

Dans cet exemple, chaque fois qu'une touche est enfoncée dans un champ de texte, la touche sera affichée dans la console.

📑 Exemple : Gestionnaire d'événements sur un formulaire


          const form = document.getElementById("form");
          
          form.addEventListener("submit", (event) => {
              event.preventDefault(); // Empêche l'envoi du formulaire
              console.log("Formulaire soumis");
          });

Dans cet exemple, l'envoi du formulaire est annulé et un message est affiché dans la console lorsque l'utilisateur soumet le formulaire.

⏳ Délai d'attente d'un événement (debounce)

Il peut être utile de limiter la fréquence à laquelle un événement est exécuté, surtout pour des événements qui se produisent fréquemment comme scroll ou resize. Cela se fait souvent avec une technique appelée debounce.


          let timeout;
          window.addEventListener("resize", () => {
              clearTimeout(timeout);
              timeout = setTimeout(() => {
                  console.log("Fenêtre redimensionnée");
              }, 300); // Exécuter après un délai de 300 ms
          });

Dans cet exemple, l'événement resize est écouté, mais l'action de redimensionnement n'est effectuée qu'après 300 ms sans changement de taille (afin de limiter les appels excessifs).

⚠️ Détacher un événement

Il est aussi possible de retirer un gestionnaire d'événements à l'aide de removeEventListener().


          button.removeEventListener("click", handler);

🔄 Propagation des événements

Les événements se propagent à travers la page, passant par des phases appelées bubbling (montée) et capturing (descente). Par défaut, un événement commence à l'élément cible et se propage vers le parent (bubbling).

Exemple de gestion des phases de propagation :


          document.getElementById("parent").addEventListener("click", () => {
              alert("Clic sur parent");
          }, true); // Capturing phase (true)
              
          document.getElementById("child").addEventListener("click", () => {
              alert("Clic sur enfant");
          }, false); // Bubbling phase (false, ou omis)

Dans cet exemple, l'événement "click" sur l'élément enfant va d'abord être capturé par le parent avant d'atteindre l'enfant.

API avancées

JavaScript propose de nombreuses API (Application Programming Interface) qui permettent d'ajouter des fonctionnalités avancées à vos applications web. Ces API vous permettent d'interagir avec des services web, manipuler le DOM, gérer les fichiers, et bien plus encore.

🔑 Qu'est-ce qu'une API en JavaScript ?

Une API en JavaScript est un ensemble d'interfaces ou de méthodes qui permettent à un script de communiquer avec le navigateur ou avec des services externes. Cela permet d'effectuer des actions comme récupérer des données depuis un serveur, manipuler des éléments DOM, ou encore interagir avec le matériel de l'utilisateur (comme la géolocalisation).

🌐 API Fetch

L'API fetch() permet de faire des requêtes HTTP asynchrones pour récupérer des données depuis un serveur web. C'est une alternative moderne à l'ancienne méthode avec XMLHttpRequest.


          // Exemple de requête GET avec fetch()
          fetch("https://api.example.com/data")
            .then(response => {
              if (!response.ok) {
                  throw new Error("Erreur réseau");
              }
              return response.json();  // Convertir la réponse en JSON
            })
            .then(data => console.log(data))  // Traiter les données
            .catch(error => console.error("Erreur:", error));  // Gérer les erreurs

Dans cet exemple, on envoie une requête GET pour récupérer des données sous format JSON depuis un serveur. La réponse est ensuite convertie et manipulée en utilisant les promesses (Promise) pour un traitement asynchrone.

📥 POST avec fetch()

Vous pouvez également envoyer des données vers le serveur en utilisant la méthode POST avec l'API fetch().


          fetch("https://api.example.com/submit", {
              method: "POST",
              headers: {
                  "Content-Type": "application/json"
              },
              body: JSON.stringify({ name: "Jean", age: 30 })
          })
            .then(response => response.json())
            .then(data => console.log(data))
            .catch(error => console.error("Erreur:", error));

Dans cet exemple, on envoie un objet JSON contenant un nom et un âge vers le serveur, qui peut être utilisé pour enregistrer des informations dans une base de données.

💻 API de Géolocalisation

L'API de géolocalisation permet d'obtenir la position géographique d'un utilisateur. Cela peut être utilisé pour des applications de cartes, de localisation ou d'autres services personnalisés en fonction de l'emplacement de l'utilisateur.


          if (navigator.geolocation) {
              navigator.geolocation.getCurrentPosition(function(position) {
                  console.log("Latitude: " + position.coords.latitude);
                  console.log("Longitude: " + position.coords.longitude);
              });
          } else {
              console.log("La géolocalisation n'est pas supportée par votre navigateur.");
          }

Dans cet exemple, on utilise navigator.geolocation.getCurrentPosition() pour récupérer les coordonnées GPS actuelles de l'utilisateur. Cette méthode est très utile pour des applications de cartographie ou des services personnalisés basés sur la localisation.

💾 API FileReader

L'API FileReader permet de lire des fichiers locaux (comme des images ou des documents) sélectionnés par un utilisateur à partir de son système de fichiers.


          const inputFile = document.getElementById("fileInput");
          
          inputFile.addEventListener("change", function(event) {
              const file = event.target.files[0];
              const reader = new FileReader();
              
              reader.onload = function(e) {
                  console.log("Fichier chargé: ", e.target.result);
              };
              
              reader.readAsDataURL(file);  // Lire le fichier comme une URL de données
          });

Dans cet exemple, on permet à l'utilisateur de sélectionner un fichier via un input de type file, et une fois le fichier sélectionné, on l'affiche sous forme d'URL de données avec FileReader.

💬 WebSockets

Les WebSockets permettent de créer une connexion persistante entre le serveur et le client pour échanger des données en temps réel. Cela est particulièrement utile pour des applications comme les chats en ligne ou les jeux multijoueurs.


          const socket = new WebSocket("ws://example.com/socket");
          
          socket.onopen = function() {
              console.log("Connexion ouverte");
              socket.send("Bonjour serveur!");
          };
          
          socket.onmessage = function(event) {
              console.log("Message reçu : " + event.data);
          };
          
          socket.onerror = function(error) {
              console.error("Erreur WebSocket : " + error);
          };
          
          socket.onclose = function() {
              console.log("Connexion fermée");
          };

Dans cet exemple, un WebSocket est ouvert vers le serveur, et l'application envoie et reçoit des messages en temps réel. Cela permet une communication bidirectionnelle continue.

🧑‍💻 API Notification

L'API de notification permet aux sites web d'envoyer des notifications aux utilisateurs, même lorsque le site n'est pas actif ou que l'utilisateur n'est pas sur la page. C'est idéal pour des alertes ou des rappels.


          if ("Notification" in window) {
              Notification.requestPermission().then(function(permission) {
                  if (permission === "granted") {
                      new Notification("Vous avez une nouvelle alerte!");
                  }
              });
          }

Dans cet exemple, on demande la permission d'envoyer une notification à l'utilisateur. Si l'utilisateur accepte, une notification s'affiche.
